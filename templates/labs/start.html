Lab Startup Guide
  1. Provision the topology.

    Create the virtual routers, switches, and server instances inside the lab environment.

  2. Configure the VPN tunnel.

    Use the VPN configuration below to connect the site-to-site tunnel between the edge router and the core network.

  3. Verify IP connectivity.

    Ping the server and switch addresses across the tunnel.

  4. SSH to devices.

    SSH directly to the switches and servers through the VPN.

{% if vpn_lab %}
VPN Configuration Snippet 
# Example strongSwan IPsec configuration
conn netforge-vpn
    left=%defaultroute
    leftsubnet=10.0.5.0/24
    right=203.0.113.10
    rightsubnet=10.0.6.0/24
    authby=psk
    keyexchange=ikev2
    ike=aes256-sha256-modp2048
    esp=aes256-sha256
    auto=add

Replace the peer address with your lab edge router IP and apply this config on the VPN peer.

SSH Access Commands

After the VPN is established, connect using these commands.

ssh admin@10.0.5.11
ssh network@10.0.5.12
ssh ubuntu@10.0.5.21

If the session fails, ensure the VPN tunnel is up and the lab services are running.
